Openclaw实践教程

发表于 2026-03-03 | 分类于 AI |

一、说明

1.1 官方文档

适用于任何操作系统的 AI 智能体 Gateway 网关，支持 WhatsApp、Telegram、Discord、iMessage 等。

发送消息，随时随地获取智能体响应。通过插件可添加 Mattermost 等更多渠道。

1.2架构说明

  • **多渠道接入 (Channels)**：支持 WhatsApp、Telegram、Discord 等，将复杂的指令交互转化为日常的即时通讯。

  • **中枢控制 (Gateway)**：作为系统的大脑，负责状态管理、模型路由及 MCP 协议分发。

  • **技能扩展 (Skills)**：原生支持 MCP（Model Context Protocol），可无缝对接文件系统、浏览器、Shell 等上百种工具。

  • **持久化记忆 (Memory)**：交互历史与知识库均以 Markdown 或向量库形式存储于本地，实现“越用越懂你”的个性化进化。

1.3 openClaw任务处理流程

  1. 意图解析：LLM 分析指令，判断是否需要调用外部工具。

  2. 工具调度：Gateway 通过 MCP 协议激活对应的本地 Skill。

  3. 反馈与记忆：执行结果反馈给 LLM，并同步更新本地记忆库，完成任务闭环。

常见问题

  • 快速排错提示

未检测到应用连接信息，请确保长连接建立成功后再保存配置 -> openclaw需要先配置appid跟app secret

onboard 卡住或报错 → 检查网络，重新运行 openclaw-cn onboard。

gateway 启动失败 → 检查端口 18789 是否被占用，可加参数 –port 其他端口。

飞书不回复 → 确认选了“长连接”、事件已订阅、App ID/Secret/Key 无误。

模型无响应 → 确认百炼 API Key 有效、配额充足。

提示飞书用户无权限->命令行进行授权 openclaw pairing approve feishu 你的飞书用户
